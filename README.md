## 💼 Projeto Destaque

### 🏥 Sistema de Painéis Hospitalares

**Plataforma completa de monitoramento operacional em tempo real** desenvolvida do zero para o Hospital Anchieta Ceilândia, com **25+ dashboards especializados** exibidos em TVs e monitores por todo o hospital.

#### 📊 Arquitetura do Sistema

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Oracle Tasy   │────▶│   Apache Hop     │────▶│   PostgreSQL    │
│   (HIS Source)  │     │   (ETL ~10min)   │     │ (Data Warehouse)│
└─────────────────┘     └──────────────────┘     └────────┬────────┘
                                                          │
                        ┌─────────────────────────────────┼─────────────────────────────────┐
                        │                                 │                                 │
                        ▼                                 ▼                                 ▼
               ┌─────────────────┐               ┌─────────────────┐               ┌─────────────────┐
               │  Flask/Python   │               │    Metabase     │               │   ntfy.sh       │
               │   (Backend)     │               │   (BI Reports)  │               │ (Notifications) │
               └────────┬────────┘               └─────────────────┘               └─────────────────┘
                        │
          ┌─────────────┼─────────────┐
          │             │             │
          ▼             ▼             ▼
     ┌─────────┐   ┌─────────┐   ┌─────────┐
     │ Desktop │   │  Mobile │   │ TV/PWA  │
     │ Browser │   │   App   │   │ Display │
     └─────────┘   └─────────┘   └─────────┘
```

#### 🎯 Painéis Implementados (25+)

| Área | Painéis | Funcionalidades |
|------|---------|-----------------|
| **Pronto Socorro** | P3, P10, P17, P18, P25 | Médicos ativos, tempo de espera, produtividade médica, exames |
| **Internação** | P4, P8, P11, P12, P22 | Ocupação hospitalar, situação pacientes, gestão de leitos |
| **Radiologia** | P19, P20 | Pendências de exames (internados e PS) |
| **Cirurgias** | P5 | Programação cirúrgica com status de workflow |
| **Farmácia** | P24 | Estoque por dias de cobertura, alertas |
| **Nutrição** | P13 | Prescrições dietéticas pendentes |
| **Faturamento** | P21 | Ciclo de contas, evolução de faturamento |
| **Ambulatório** | P23 | Tempo de espera e atendimentos |
| **TI/Helpdesk** | P14, P15 | Chamados ativos, histórico, SLA |
| **Recepção** | P16 | Performance de atendentes |
| **IA Clínica** | P6, P7 | Priorização de risco (NEWS2/SOFA), detecção de sepse |

#### 🔧 Funcionalidades Técnicas

- ⏱️ **Atualização em tempo real** — Refresh automático a cada 60s
- 📺 **Auto-scroll inteligente** — Com watchdog para displays em TV
- 🔐 **Sistema de permissões granular** — Controle por painel e usuário
- 🔔 **Notificações push** — Alertas via ntfy.sh para admissões, pareceres, prescrições
- 📱 **PWA** — Instalável em dispositivos móveis
- 🤖 **Análise IA** — Score de risco clínico automatizado (Groq/Llama 3.3)
- 📈 **ETL automatizado** — Pipelines Apache Hop com ciclos de ~10 minutos
- 📊 **BI integrado** — Metabase conectado ao data warehouse
- 🩺 **Health checks** — Monitoramento via Uptime Kuma

#### 🏗️ Stack do Projeto

| Camada | Tecnologias |
|--------|-------------|
| **Frontend** | HTML5, CSS3, JavaScript ES5, Font Awesome |
| **Backend** | Python 3.8+, Flask 3.0, psycopg2, bcrypt |
| **Database** | PostgreSQL 12+ (DW), Oracle DB (Tasy HIS) |
| **ETL** | Apache Hop (TRUNCATE + INSERT pattern) |
| **BI** | Metabase |
| **IA** | Groq API (Llama 3.3 70B) |
| **Infra** | Windows Server 2019, NSSM, Nginx |
| **Monitoring** | Uptime Kuma (Node.js) |
| **Notifications** | ntfy.sh |
